Go Language

Script Organization

If you have any ideas on how to organize the script better please let me know. All the scripts start with cj to:

  • reduce the probablity of the system using my executable with the same name as a system executable,
  • to make it easier for the cjCmds script to find scripts that I wrote and display information about them and
  • not have a Java like full name space executable requirement, which becomes burden some when runing the script.

Script Descriptions

No attempt has been made to support the script on multiple OSes and they are used on Ubuntu. It should be strainght forward to convert scripts that don't use OS specific calls in them to run on other OSes.

ScriptDescription.
cjCmds Linux List script that start with cj and one line description of the function.
cjScreenOff Linux Turns the screen off. Can save power when walking away from a machine if the screen power down time out is long.

Reasons For Chosing Go

LanguageReason not selected.
Scala JVM based langauges (Clojure, etc), which are "better" languages than go, were not selected because:
  • Startup time is slow due to the JVM
  • Development on cloud infarstructure would cost more due to the JVM extra CPU cycle usage during development when the JVM restarted very frequently. The cost for RAM usages will also be higher due the JVM.
Rust Rust run faster and the base language syntax is better than Go but Rust was not selected for the following reasons:
  • The concurrency model in Go is simpler to use in practice.
  • Rust does not have as many libraries, frameworks and third party support, as Go does, as of February 2016.
All All languages share the following short coming relative to GO:
  • Google is developing GO for other domains that other language developers do not have the resources to presue. Some of the domains that Google is presuing with GO are:
    • Android development with Go. Google already has released tools to code Android application in Go (February 2016).
    • Google Cloud application development. Google has tools and documentation for writting application for Google's cloud infrastructure. The Go langauge team is working with the Cloud team to make sure the language evolves to make Cloud development easire.
